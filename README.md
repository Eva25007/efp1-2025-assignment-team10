## 1. Περιγραφή
Σύντομη περιγραφή του σεναρίου που υλοποιείται:
Η εφαρμογή υλοποιεί ένα σύστημα ηλεκτρονικής συνταγογράφησης που επιτρέπει σε ιατρούς να εκδίδουν συνταγές, και σε φαρμακοποιούς να επαληθεύουν και να εκτελούν συνταγές.

Βασικοί χρήστες (Ρόλοι):
Ιατρός: Δημιουργεί και εκδίδει συνταγές
Φαρμακοποιός: Επαληθεύει και εκτελεί συνταγές

Υλοποιημένα Use Cases:
### Ιατρός
- Καταχώρηση Συνταγής (main)
- Αναζήτηση/Επιλογή Φαρμάκου (include)
- Παράδοση Συνταγής (include)

### Φαρμακοποιός
- Εκτέλεση Συνταγής (main)
- Επαλήθευση Φαρμάκου (include)

## 5. Δομή του κώδικα
Σύντομη περιγραφή των αρχείων:

```text
efp1-2025-assignment-team10/
.
├── main.py         # Κεντρικό σημείο εκκίνησης της εφαρμογής
├── models.py       # Ορισμός κλάσεων (Doctor, Patient, Prescription, Drug, Pharmacist κ.ά.)
├── services.py     # Λογική συστήματος και υλοποίηση των use cases
└── diagrams/
    ├── class_diagram.png
    ├── sequence_diagram1.png
    └── sequence_diagram2.png

```

Περιγραφή αρχείων:

main.py

Υλοποιεί το κεντρικό μενού της εφαρμογής, διαχειρίζεται την αλληλεπίδραση με τον χρήστη (input/output) και καλεί τις αντίστοιχες συναρτήσεις από το services.py.

models.py

Περιέχει τις βασικές κλάσεις του συστήματος (οντότητες), όπως Ιατρός, Ασθενής, Φαρμακοποιός, Συνταγή και Φάρμακο, καθώς και τα βασικά τους πεδία.

services.py

Περιλαμβάνει τη λογική των use cases (δημιουργία συνταγής, αναζήτηση φαρμάκου, επαλήθευση και εκτέλεση συνταγής), καθώς και τις συναρτήσεις που συνδέουν τα δεδομένα με τη ροή της εφαρμογής.

diagrams/

Περιέχει τα UML διαγράμματα (Class και Sequence) που τεκμηριώνουν τη σχεδίαση του συστήματος.
