## 1. Περιγραφή
Σύντομη περιγραφή του σεναρίου που υλοποιείται:
Η εφαρμογή υλοποιεί ένα σύστημα ηλεκτρονικής συνταγογράφησης που επιτρέπει σε ιατρούς να εκδίδουν συνταγές, και σε φαρμακοποιούς να επαληθεύουν και να εκτελούν συνταγές.

##Βασικοί χρήστες (Ρόλοι):
Ιατρός: Δημιουργεί και εκδίδει συνταγές
Φαρμακοποιός: Επαληθεύει και εκτελεί συνταγές

##Υλοποιημένα Use Cases:
Καταχώρηση Συνταγής
Αναζήτηση/Επιλογή Φαρμάκου
Εκτέλεση Συνταγής
Επαλήθευση Φαρμάκου

## 8. Γνωστά Περιοριστικά Σημεία / Μελλοντική Επέκταση
Τρέχοντα Περιοριστικά:
Δεν υλοποιήθηκε το Use Case "Προβολή Ιστορικού Συνταγών" για τον ασθενή
Τα δεδομένα αποθηκεύονται μόνο στη μνήμη (χάνονται με το κλείσιμο του προγράμματος)
Δεν υπάρχει πλήρης έλεγχος εγκυρότητας για όλα τα πεδία εισαγωγής
Δεν υλοποιήθηκαν όλες οι μη λειτουργικές απαιτήσεις (κρυπτογράφηση, auto-logout)

Μελλοντική Επέκταση:
Υλοποίηση των use cases που λείπουν (Ιστορικό)
Αποθήκευση δεδομένων σε βάση δεδομένων
Ενσωμάτωση κρυπτογράφησης για ευαίσθητα δεδομένα
Προσθήκη συστήματος αυθεντικοποίησης χρηστών
Υποστήριξη για πολλαπλούς ταυτόχρονους χρήστες
Δημιουργία web διεπαφής χρήστη
Ενσωμάτωση API για επικοινωνία με εξωτερικά συστήματα (φαρμακεία, ασφαλιστικά ταμεία)
